1. Introduction

1.1 Overview

The Mobile Application is designed to simplify and digitize workshop and garage operations through a mobile-first platform. The application enables workshop staff to efficiently manage repair orders, inspections, customer communication, appointments, service workflow tracking, and payment monitoring.

The application focuses on improving operational efficiency, reducing manual work, and providing quick access to workshop activities directly from mobile devices.

1.2 Objectives

Simplify daily workshop operations

Digitize repair order creation process

Improve customer communication management

Enable faster vehicle inspection workflows

Improve service workflow tracking

Provide real-time operational visibility

Reduce manual data entry using automation and AI features

1.3 Target Users

Garage Owners

Service Advisors

Technicians

Billing Staff

2. Login &amp; Authentication Module

2.1 Feature Overview

The Login &amp; Authentication module allows registered users to securely access the application using username and password credentials.

2.2 Available Features

Username Input Field

Password Input Field

Sign In Button

Forgot Password Option

Create Account Option

Secure Session Authentication

2.3 Login Flow

User opens the mobile application.

User enters registered username.

User enters password.

User clicks on the Sign In button.

System validates credentials.

On successful authentication, user is redirected to Dashboard/Home screen.

2.4 Forgot Password Flow

User clicks on Forgot Password.

System redirects user to password recovery screen.

User enters registered mobile number or email address.

OTP verification process is completed.

User creates new password.

User logs in using updated password.

2.5 Create Account Flow

User clicks on Create Account.

User enters required registration details.

User submits registration form.

System validates entered details.

Account is created successfully.

User logs in using newly created credentials.

2.6 Validations

Username field should not be empty

Password field should not be empty

Invalid credentials should display error message

Password should satisfy security policy requirements

2.7 Expected Result

Registered users should log in successfully

New users should create accounts successfully

Password recovery should complete securely

4.2 Create RO Workflow

4.3 Vehicle Number Detection

4.3.1 Description

The RO creation process starts with vehicle number detection.

Users can:

Capture the vehicle number plate image

Enter vehicle number manually

The application integrates with:

Vahan API

to fetch vehicle details automatically.

4.3.2 Available Features

Number Plate Camera Capture

Manual Vehicle Number Entry

Auto Number Detection

Vehicle Details Fetching

Next Button

Back Button

Close Button

4.3.3 Workflow

User clicks on Create RO.

Camera screen opens.

User captures vehicle number plate image.

The system detects the vehicle registration number automatically.

Vehicle number is populated in input field.

The user can manually enter the vehicle number.

Vehicle details are fetched using MMVY/Vahan APIs.

User clicks on Next.

4.3.4 Expected Result

The vehicle number should be detected successfully

Vehicle details should fetch successfully

Manual entry should work correctly

12. Summary

The Mobile Application provides a complete digital workflow solution for workshop and garage management. The application simplifies repair order creation, inspection handling, customer communication, appointment management, vehicle tracking, and payment monitoring through an integrated mobile platform.

The platform improves operational visibility, reduces manual processes, enhances service efficiency, and enables better customer experience management for workshop operations.
